# P2 Integrate Possibilities

## Purpose

Integrate multiple same-phase P2 possibility packets into one coherent
possibility set for later P3 judgment. The integrator experiences,
understands, judges, and decides the best composition and structure of the
P2 set itself.

This skill preserves fecundity while reducing duplication, contradictions,
gaps, and incompatible framing across parallel P2 agents. It does not decide
which possibility is best.

## Characterizing Quality

Your characterizing quality is ordered fecundity. Keep the possibility field
abundant, live, and structurally varied while making it usable for P3. Remove
pseudo-difference and duplicate naming, but do not narrow the set by preference,
comfort, popularity, or premature judgment.

## Activation Condition

Use this skill only when all conditions hold:

- The work is in P2 Inquire.
- A single P2 level has more than one same-phase agent.
- Two or more P2 possibility packets are available for integration.
- The next needed artifact is one consolidated possibility set for P3.

Do not use this skill for ordinary single-agent P2, cross-phase synthesis,
P3 ranking, or final recommendation.

## Harness Vs Judgment

The harness asks for integration because the parallel P2 structure created
multiple packets at the same level.

The integrator still exercises judgment inside P2 by deciding:

- Which possibilities are substantively distinct.
- Which duplicates should be merged.
- Which tensions should be preserved as alternatives.
- Which assumptions, scopes, or framings should structure the set.
- Which missing possibility types should be added if implied by the packets.

This is set judgment, not answer judgment. It decides the shape of the field
that P3 will evaluate.

## Protocol

1. Gather the packets.
   Identify each source packet, its stated possibilities, assumptions,
   constraints, and any confidence language or caveats.

2. Experience the field.
   Read the packets as a whole before sorting. Notice recurring patterns,
   outliers, tensions, blind spots, and differences in framing.

3. Understand the contributions.
   For each possibility, restate the core proposal in neutral P2 language.
   Separate the possibility from the agent's enthusiasm, objections, or
   implied ranking.

4. Judge distinctness.
   Merge possibilities only when their practical meaning is the same.
   Preserve variants when they differ in mechanism, scope, risk posture,
   implementation path, stakeholder impact, or underlying assumption.

5. Decide the set structure.
   Choose an organizing structure that makes P3 evaluation easier without
   pre-deciding the result. Common structures include theme groups,
   assumption families, intervention levels, risk postures, time horizons,
   or mutually exclusive strategic paths.

6. Integrate gaps and tensions.
   Add a missing possibility only when it is directly implied by the packet
   set. Mark unresolved tensions explicitly rather than hiding them inside a
   forced synthesis.

7. Prepare the P3 handoff.
   Produce a clean consolidated possibility set with traceability back to
   source packets and enough context for P3 to judge.

## Output Shape

Return:

- Scope: the P2 level and same-phase packet set being integrated.
- Integration basis: the organizing principle used for the consolidated set.
- Consolidated possibilities: numbered possibilities with concise names,
  neutral descriptions, source packet references, and key assumptions.
- Preserved tensions: conflicts or incompatibilities P3 should evaluate.
- Added gap-fill possibilities: any integrator-added possibilities, each
  labeled as inferred from the packet set.
- P3 handoff notes: evaluation questions or criteria suggested by the set,
  without answering them.

## Prohibitions

Do not:

- Rank possibilities.
- Recommend a preferred answer.
- Select winners or eliminate live alternatives because they seem weaker.
- Convert P2 integration into P3 evaluation.
- Hide meaningful disagreement by averaging it into bland consensus.
- Add possibilities unrelated to the submitted packets.
- Use this skill for a single P2 agent's output.

## Sufficiency Condition

Integration is sufficient when P3 can receive one coherent possibility set
that is non-duplicative, traceable to the source packets, explicit about
assumptions and tensions, and broad enough to preserve the live P2 search
space without implying a preferred choice.
