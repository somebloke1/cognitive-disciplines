# P2 Inquire Possibilities

## Purpose

Use this skill to perform P2: intelligence as Fecundity. Your job is to generate multiple structurally distinct possibilities grounded in P1's characterization. P2 presents abundance to judgment; it does not select.

For a compact source-grounding summary, read [references/ep-discipline.md](references/ep-discipline.md) when the task concerns EP audit or curriculum-derived cognitive-cycle source materials.

For team, legion-scale, or full-cycle work, use the shared packet contract at [../full-cognitive-cycle/references/cognitive-cycle-packet-contract.md](../full-cognitive-cycle/references/cognitive-cycle-packet-contract.md). P2 produces a P2 packet grounded in P1; it does not judge or decide.

## Characterizing Quality

Your characterizing quality is expansive Fecundity. Generate abundance for
reason: multiple live possibilities that differ in structure, assumptions,
strategy, causal account, or development path. Include uncomfortable
possibilities when P1 supports them. Do not satisfy P2 with renamed versions of
one idea.

## Inputs

Start from a P1 characterization or an equivalent evidence bundle. If the task lacks enough concrete evidence, say that P2 is provisional and identify the evidence needed to make possibilities non-speculative.

## Harness vs Judgment

The soft harness supplies the P1 packet, cycle manifest, output packet shape, and handoff target. P2 judgment decides what counts as a structurally distinct possibility, which assumptions are live, and what evidence would differentiate alternatives. A list is not fecundity unless the alternatives differ in underlying structure.

## Protocol

1. Extract generative tensions from P1.
   - Contradictions, absences, unstable interfaces, brittle dependencies, proposal/substrate mismatches, recurring breakdowns, and unexplained successes.

2. Generate 3-7 structurally distinct possibilities.
   - Vary the underlying scheme, causal account, implementation topology, governance move, or emergence path.
   - Avoid surface variants of the same idea.
   - Include at least one uncomfortable or disruptive possibility when the evidence supports it.

3. Ground each possibility.
   - Cite the P1 findings that motivate it.
   - State the assumptions required.
   - Identify what would become possible if it were true.
   - Identify what data would distinguish it from the others.

4. Apply the EP lens when relevant.
   - Possibilities may concern scheme health, conditioned-series ordering, probability of emergence, probability of survival, breakdown cascades, blind alleys, defenses, floating-population components, or proposal verdict paths.

5. Check for surface variants in team or legion-scale cycles.
   - Collapse duplicate possibilities that differ only in wording.
   - Preserve genuinely conflicting alternatives as distinct, especially uncomfortable or disruptive ones.
   - Mark any possibility that requires further P1 evidence before P3 can judge it.

## Output Shape

For each possibility include:

- Name
- Core claim
- Grounding in P1 evidence
- Assumptions
- Consequences if true
- Differentiating evidence or tests

End with:

- Surface-Variant / Duplicate Check
- Questions for P3, only if there are specific judgment tensions

Do not rank the possibilities.

## Discipline

Do not decide which possibility is best. Do not hide a single favored interpretation inside a list. Do not invent possibilities disconnected from P1. A P2 pass is sufficient when P3 has real alternatives to evaluate.
