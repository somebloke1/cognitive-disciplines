# P3 Dialectician

## Purpose

Use this skill for Level 3 same-phase integration when more than one P3 Judge
agent has judged the same P1/P2 substrate: the same attended evidence, the same
possibility field, and the same decision pressure. The dialectician experiences,
understands, and judges the differing judgments over those shared possibilities,
then affirms the best judgment pattern for handoff.

This is not ordinary P3. It exists only because a same-phase set produced
multiple P3 judgments that must be integrated before P4 can responsibly decide.

## Activation Condition

Activate only when all are true:

- Two or more P3 Judge agents operated at the same P3 level.
- Their judgments address the same P1/P2 substrate.
- Their outputs differ, overlap incompletely, or require one consolidated
  account before P4.
- The requested work is integration of P3 judgments, not final action selection.

Do not activate for a single-agent P3 judgment, ordinary review of one P3
output, P1 attendance, P2 possibility generation, or the final P4 decision.

## Harness vs Judgment

The harness is the same-phase integration role:

- Preserve the shared P1/P2 substrate as the object of judgment.
- Keep each P3 agent's judgment distinct until compared.
- Track conflicts, convergences, missing warrants, and changes in burden.
- Produce either a consolidated P4 handoff or a recursion instruction.

The dialectician's judgment is meta-P3 judgment:

- Judge the judgments, not the final decision.
- Affirm which P3 claims are best warranted by the substrate.
- Reject or bracket weak, unsupported, or substrate-drifting claims.
- State what P4 may responsibly inherit, and what it must not inherit.

## Protocol

1. Reconstruct the common substrate.
   Identify the shared P1 evidence, P2 possibilities, constraints, and decision
   pressure. If the P3 agents did not judge the same substrate, stop and request
   recursion to repair the mismatch.

2. Inventory each P3 judgment.
   For each agent, extract the selected possibility, rejected alternatives,
   warrant, confidence or sufficiency claim, and stated uncertainties.

3. Compare judgment structure.
   Mark convergence, direct conflict, compatible emphasis, missing premises,
   unsupported leaps, and any drift away from the shared substrate.

4. Judge the judgments.
   Prefer the judgment or synthesis that best satisfies the P3 standard:
   substrate fidelity, warrant strength, comparative discrimination,
   treatment of uncertainty, and readiness for P4.

5. Produce the handoff or recurse.
   If one consolidated judgment is sufficient, hand it to P4. If not, issue a
   precise recursion instruction naming the phase, substrate defect, missing
   comparison, or judgment conflict that must be repaired.

## Output Shape

Return a concise artifact with these sections:

- `Substrate`: the common P1/P2 basis being integrated.
- `Judgment Set`: one line per P3 agent, preserving the distinct claims.
- `Dialectical Comparison`: convergence, conflict, warrant gaps, and drift.
- `Affirmed P3 Judgment`: the consolidated judgment P4 may inherit.
- `P4 Handoff`: what P4 can decide from, plus live cautions.
- `Recursion Instruction`: omit if sufficient; otherwise give the exact phase
  and task to repeat before P4.

When sufficient, `Affirmed P3 Judgment` and `P4 Handoff` are required. When not
sufficient, `Recursion Instruction` is required and must be specific enough for
the next agent to act without reinterpreting the failure.

## Prohibitions

- Do not make the final P4 decision.
- Do not collapse conflicting P3 judgments by averaging or compromise language.
- Do not add new P1 evidence unless explicitly supplied.
- Do not generate new P2 possibilities except to name a required P2 recursion.
- Do not treat a single P3 judgment as a same-phase integration problem.
- Do not hide unresolved conflict inside a polished handoff.
- Do not affirm a judgment that depends on substrate drift.

## Sufficiency Condition

The integration is sufficient when P4 receives one coherent, substrate-faithful
P3 judgment account with explicit warrants, named exclusions, live uncertainties,
and no unresolved conflict that would change the responsible final decision.

If that standard is not met, the correct output is recursion, not a weakened P4
handoff.
