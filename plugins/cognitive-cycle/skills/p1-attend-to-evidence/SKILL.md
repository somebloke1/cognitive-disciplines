# P1 Attend To Evidence

## Purpose

Use this skill to perform P1: attentiveness as Growth. Your job is to characterize what is given with high fidelity so later operations can inquire, judge, and decide without being starved or biased.

For a compact source-grounding summary, read [references/ep-discipline.md](references/ep-discipline.md) when the task concerns EP audit or curriculum-derived cognitive-cycle source materials.

For team, legion-scale, or full-cycle work, use the shared packet contract at [../full-cognitive-cycle/references/cognitive-cycle-packet-contract.md](../full-cognitive-cycle/references/cognitive-cycle-packet-contract.md). P1 produces a P1 packet; it does not own the later cycle.

## Harness vs Judgment

The soft harness supplies scope, packet shape, orienting question, implicit unknown, source scope, and handoff target. P1 judgment decides what evidence is relevant, what is directly observed, what is absent, and what is only an inference. A well-shaped packet does not excuse weak attention.

## Protocol

1. Stabilize the scope and authority.
   - Identify the user request, artifacts, repository path, proposal, orienting question, and implicit unknown if present.
   - Prefer primary local sources over memory or paraphrase.
   - For curriculum-derived EP audit work, inspect the relevant grounding sources before task interpretation when available: source theory, operating instructions, constitutions, root prompts, role definitions, curricula, rubrics, task prompts, and current material records.

2. Gather before interpreting.
   - Read the files, logs, prompts, tool definitions, tests, schemas, and prior reports that condition the task.
   - Capture concrete facts: names, paths, states, interfaces, schemas, stages, verdict values, fields, dependencies, and absences.
   - Mark any claim that goes beyond direct observation as an inference.

3. Characterize EP-audit-relevant material.
   - Functioning schemes of recurrence.
   - Conditioning relationships among schemes; explicitly name conditioning relationships in the evidence.
   - Defenses, records, recovery loops, validation gates, and lifecycle boundaries.
   - Breakdowns, blind alleys, brittle points, and missing conditions.
   - Emergence candidates and proposal substrate if a proposal is under review.
   - Recent trajectory when logs or records are part of the evidence.

4. Preserve negative space.
   - Name what you looked for and did not find.
   - Distinguish "absent from inspected sources" from "absent from the project".
   - Note unresolved contradictions without resolving them.

5. Produce a P1 packet when in a cycle.
   - Include the common packet header when a cycle manifest exists.
   - Include evidence anchors precise enough for P2/P3 to inspect.
   - Include risks of P1 insufficiency: uninspected sources, weak evidence, stale data, contradictions, or scope limits.
   - Handoff only candidate significance signals to P2, not recommendations.

## Output Shape

Use sections like:

- Scope and Sources Inspected
- Observations
- Relevant Absences
- EP Scheme Map
- Conditioning and Dependencies
- Breakdowns, Blind Alleys, and Defenses
- Inferences and Uncertainties
- Risks of P1 Insufficiency
- Material for P2

## Discipline

Do not recommend, rank, select, or decide. Do not compress away concrete evidence into generic summaries. If you suspect an interpretation, label it as a candidate inference for P2 rather than smuggling it into the characterization as fact.

P1 is sufficient when another agent could use only your characterization to generate grounded possibilities without rereading the entire substrate.
