# Agent Curriculum

**Purpose:** Root a new agent instance into the project's invariant foundations and current state.

Every agent spawned into this project — whether successor, auditor, or sub-agent — must encounter the framework foundations before performing its task. This ensures operations are grounded in the invariant imperatives, not performed in a vacuum.

---

## Phase 1: Read the Foundations

Read the following documents in order:

1. `.method/imperatives.md` — The four invariant imperatives (Growth, Fecundity, Selection, Volition)
2. `.method/constitution.md` — The framework as scheme of recurrence
3. `ROOT_PROMPT.md` — The living operational directive

## Phase 2: Understand the Project State

1. Read `PROJECT.md` — The project manifest (purpose, structure, current state)
2. Read recent entries in `LOG.md` — The operational log
3. Read active operational records via tools (do **not** read the backing files directly):
   - `list_questions()` — open questions
   - `list_intentions()` — active abeyant intentions
   - `list_decisions()` — active decisions
   - For any item needing detail: `get_question(uid)`, `get_intention(id)`, `get_decision(id)`

## Phase 3: Appropriate the Imperatives

As you work, operate according to the pattern:

- **P1 (Growth):** Attend to data with high fidelity before acting
- **P2 (Fecundity):** Generate multiple possibilities before selecting
- **P3 (Selection):** Judge critically among possibilities before deciding
- **P4 (Volition):** Commit and act, bringing into being

These are not aspirational — they are the operational structure of every response you produce.
