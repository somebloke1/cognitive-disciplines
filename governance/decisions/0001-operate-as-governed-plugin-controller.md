# Decision 0001: Operate As Governed Plugin-Development Controller

- `status`: accepted
- `date`: 2026-06-21
- `github`: https://github.com/somebloke1/cognitive-disciplines/issues/4

## Decision

Operate as a governed plugin-development controller for the persistent objective
to develop the cognitive cycle into a full-fledged Codex plugin.

## Rationale

The requested end state is not merely a collection of skills. It requires git,
GitHub, commits, issues, PRs, GitHub project-surface tracking, local governance
records, and a disciplined distinction between deterministic harness mechanics
and language-model cognitive judgment.

## Implications

- Develop practices and norms before plugin transformation.
- Use GitHub issues and the GitHub Project board as the external tracking surface.
- Use local governance ledgers for decisions, intentions, open questions, and tasks.
- Commit durable governance and implementation artifacts.
- Use PRs for externally reviewable implementation milestones.
- Preserve noetic-pi evidence as source material, but translate it into Codex-native plugin mechanics.

## Non-Goals

- Do not treat prompt labels as sufficient cognitive discipline.
- Do not implement the plugin before acceptance gates and controller practices exist.
- Do not collapse deterministic harness mechanics into LLM judgment.
